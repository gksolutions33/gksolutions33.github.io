A Comprehensive Guide to Building React Apps with TypeScript

Creating dynamic web applications has never been easier with technologies like React and TypeScript. As developers, we often seek efficient tools that enhance our workflow and code quality, making TypeScript in conjunction with React an optimal pairing. This article explores how to build React applications using TypeScript, illustrating the process through a practical project—an advanced todo list app called TaskFi.

What is TypeScript?

TypeScript is a superset of JavaScript that introduces static types, enabling developers to catch errors during development instead of runtime. It enhances readability and maintainability, making it especially suitable for large-scale applications. TypeScript promises a smoother development experience and provides robust tooling.

Why Use TypeScript with React?

  1. Error Reduction: TypeScript catches potential issues early in the development process.
  2. Better IDE Support: Enhanced auto-completion and code navigation features.
  3. Maintainability: Helps manage types in large applications, making it easier to refactor code later.
  4. Scalability: Perfect for building complex applications due to type safety and clear code structure.

Implementing TypeScript in Components

Setting Up the Input Field

In InputField.tsx, we will create a functional component to accept input for new tasks. We will use the useState hook to manage our input state and pass props for better type management.

import React, { useState } from 'react';

const InputField: React.FC<{ setTodo: (todo: string) => void }> = ({ setTodo }) => {
    const [input, setInput] = useState<string>('');

    const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => {
        setInput(e.target.value);
    };

    const handleSubmit = (e: React.FormEvent) => {
        e.preventDefault();
        if (input.trim()) {
            setTodo(input);
            setInput('');
        }
    };

    return (
        <form onSubmit={handleSubmit}>
            <input type="text" value={input} onChange={handleChange} placeholder="Enter a task" />
            <button type="submit">Go</button>
        </form>
    );
};

export default InputField;

Building the Todo List

Next, we'll create the TodoList component to manage our collection of task items. This component interacts with the main app state and passes necessary props down to each SingleTodo component.

import React from 'react';
import SingleTodo from './SingleTodo';

const TodoList: React.FC<{ todos: Todo[], setTodos: React.Dispatch<React.SetStateAction<Todo[]>> }> = ({ todos, setTodos }) => {
    return (
        <div>
            {todos.map((todo, index) => (
                <SingleTodo key={todo.id} todo={todo} todos={todos} setTodos={setTodos} index={index} />
            ))}
        </div>
    );
};

export default TodoList;

Adding the Logic to Manage Tasks

Now, we will implement functions to add, edit, delete, and complete tasks. Each of these will manipulate the todos state and utilize TypeScript interfaces to ensure type safety.

interface Todo {
    id: number;
    title: string;
    done: boolean;
}

const handleAdd = (title: string) => {
    const newTodo: Todo = { id: Date.now(), title, done: false };
    setTodos(prev => [...prev, newTodo]);
};

const handleDelete = (id: number) => {
    setTodos(prev => prev.filter(todo => todo.id !== id));
};

const handleToggle = (id: number) => {
    setTodos(prev =>
        prev.map(todo => (todo.id === id ? { ...todo, done: !todo.done } : todo))
    );
};
